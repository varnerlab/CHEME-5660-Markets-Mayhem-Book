Working with Data in Julia#

Introduction#

Working with data is one of the most common tasks in any programming language, including Julia. For example, loading data into a program, doing some calculations using that data, and then saving the results to a file (either locally or in the cloud) describes the majority of programs you will write in this course.

There are various tools and approaches to reading and writing data files in Julia. This chapter will consider a few common approaches and introduce some patterns for working with data files. In particular, we will:

  • Introduce reading and writing Comma seperated value files, a common data format for numerical data files

  • Introduce more expressive tools such as DataFrames, which are in-memory tabular data structures

  • Introduce reading and writing data in Binary file formats, a common approach for large data files and more complex data structures

Comma seperated value files#

Arguably, the most widely used data file format for the universe of possible applications is the comma separated value (CSV) format. A CVS file is a delimited text file that uses a comma to separate data values. Each line of a CSV file is called a record, where each record consists of one or more fields (data values), separated by commas; hence the name comma separated value file.

A CSV file is typically used to store tabular data (numbers and text) in plain text, where each line has the same number of fields. However, the CSV file format is not standardized. Thus, while separating fields in records with commas is standard, various exceptions and exceptional cases sometimes make CSV files problematic, e.g., fields with commas in the data or records with a variable number of fields can make working CSV files challenging.

Fortunately, the applications we are interested in typically involve numerical CSV files that are easily handled using the CSV.jl package. The CSV.jl package can be downloaded and installed using the Julia package manager. To load the CSV package into your program, or the REPL, issue the using or import commands. Once loaded, the CSV package publishes several commands to read and write CSV files.

DataFrames#

DataFrame.jl is a Julia package that provides tools for working with tabular data in Julia, e.g., numerical data tables.

The DataFrame.jl package can be downloaded and installed using the Julia package manager. To load the DataFrames package into your program, or the REPL, issue the using or import commands. Once the DataFrames package is loaded, it publishes several commands to work with tabular data.

Objects of type DataFrame represent data tables where each column corresponds vector of values. The simplest way to construct a DataFrame is to pass vectors holding the values for each column using keyword arguments or pairs:

# This example was reproduced from the DataFrames.jl package documentation
# https://dataframes.juliadata.org/stable/man/basics/#First-Steps-with-DataFrames.jl

# Load external packages
using DataFrames

# build a sample DataFrame -> store in variable df
df = DataFrame(A=1:4, B=["M", "F", "F", "M"])

4 rows × 2 columns

AB
Int64String
11M
22F
33F
44M

A DataFrame can also be created from a Dictionary, where the column header names are the keys of the dictionary, and the columns are the data associated with each key:

# This example was reproduced from the DataFrames.jl package documentation
# https://dataframes.juliadata.org/stable/man/basics/#First-Steps-with-DataFrames.jl

# Load external packages
using DataFrames

# create a dictionary w/data -> variable dict
dict = Dict(:customer_age => [15, 20, 25], 
  :first_name => ["Rohit", "Rahul", "Akshat"])

# convert the dictionary to a DataFrame -> variable df
df = DataFrame(dict)

3 rows × 2 columns

customer_agefirst_name
Int64String
115Rohit
220Rahul
325Akshat

Notice the Dictionary keys are of type Symbol and not Strings; a Symbol represents an identifier in parsed julia code (ASTs). A Symbol is often used as a name or label to identify an entity, e.g., as a dictionary key as shown above. Symbols can be entered using the : quote operator or constructed from Strings using the Symbol(x...) constructor method.

Reading and writing dataframes from disk#

While it’s easy to create a DataFrame manually, many of the applications that we are interested in typically require that we read data files from disk and convert that into a DataFrame object, or we save a DataFrame object to a file.

Binary file formats#

Binary file formats have been developed to store and organize large amounts of complex structured data. The basis of many binary formats is the Hierarchical Data Format (HDF), a set of file formats initially developed by the U.S. National Center for Supercomputing Applications (NCSA).

The JLD2.jl package saves and loads Julia data structures in a format that is a subset of HDF5, the latest iteration of the Hierarchical Data Format (HDF).

The JLD2.jl package can be downloaded and installed using the Julia package manager. To load the JLD2 package into your program, or the REPL, issue the using or import commands. Once the JLD2 package is loaded, it publishes several commands to load and save data in the HDF5 format. Let’s look at the save and load functions.

save#

Consider an example of saving a JLD2 file; in this example, we downloaded stock price data for several firms from an external application programming interface (API) and saved this dataset as a Dictionary. The price history for each firm is stored in a DataFrame, while the overall dataset for N = 25 firms is stored as a Dictionary where the ticker symbol for firm XYZ (of type String) is used as the Dictionary key.

The save function accepts a path and an AbstractDict holding the key/value pairs that we wish to save, where the key is a string representing the name of the dataset and the value is its contents:

# load external packages
using DataFrames
using JLD2
using FileIO

# external process to download data (not shown)
dd = api({arguments}); # dd -> Dict{String,DataFrame}

# path to data file 
filename = "Portfolio-Data-06-20-22.jld2"

# save
save("Portfolio-Data-06-20-22.jld2", Dict("dd" => dd)); # saved dd in Dict with key dd

load#

Consider an example of loading a JLD2 file; in this example, a Dictionary holding stock market data is loaded from disk. In particular, the price data for each of N = 25 firms is stored in a DataFrame, where the ticker symbol for each firm XYZ is used as the Dictionary key; the ticker symbol XYZ is of type String. Thus, the loaded Dictionary is of type Dict{String,DataFrame}.

The load function reads a *.jld2 file from the disk. When called with a path argument only, the load function loads the dataset from the given file into a Dictionary:

# load external packages
using DataFrames
using JLD2
using FileIO

# set path to the data file
# joinpath is cross platform: handles / vs \
path_to_data_file = joinpath(".", "data", "Portfolio-Data-06-20-22.jld2")

# load the dataset: the original data is stored in a Dict with key `dd`
dd = load(path_to_data_file)["dd"] # dd -> Dict{String,DataFrame}

Taken together, the JLD2.jl package provides tools to directly persist (and load) complex hierarchical data structures, which go well beyond just numerical data. For more information, please see the JLD2.jl package documentation.
